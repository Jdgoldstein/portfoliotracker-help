🚀 Quick Start
1️⃣Open the tracker
Open the Portfolio_Tracker.html file in Chrome, Brave, or Edge. The tracker runs entirely in your browser — no installation, no server, no account required.
Tip: Chrome, Brave, and Edge all work. Brave users: set Shields to Standard so live prices load. Firefox and Safari are not recommended — File System auto-save is unsupported.
2️⃣Link your backup file
Click ⛁ Data ▾ → Link / Change File and create or select a .json file on your drive. This enables auto-save every 7 minutes and protects against browser cache clears.
3️⃣Add your first position
Go to the tab that matches what you want to track — LP Pools for liquidity positions, Vaults for staking, Holdings for spot tokens. Click the + button in each section to add entries.
4️⃣Refresh prices
Click ↻ Refresh in the header to fetch live prices from CoinGecko. Prices auto-refresh every few minutes while the tracker is open.

⛁ Data & Backup
📦Where is my data stored?
All data lives in two places: your browser's localStorage (fast, in-memory) and a linked .json file on your local drive. Nothing is sent to any external server. You own your data completely.
🔄Auto-save
When a backup file is linked, the tracker auto-saves every 7 minutes silently. The save timestamp shows in the ⛁ menu. Only your entered data is saved — live price feeds are excluded to keep files small and clean.
🛟After a cache clear
If you clear browser cache, localStorage is wiped. To recover: open the tracker → click ⛁ Data ▾ → Import / Link Backup → select your .json backup file. All data restores immediately.
Important: Keep a recent backup file. Export a dated copy via ⛁ → Save / Export Backup before any major browser cleanup.
📋Backup file format
Backup files use a versioned envelope format: { db: {...}, hpcHistory: {...}, savedAt, schemaVersion: 2 }. This keeps your portfolio data and hedge chart history together in one file, while maintaining clean separation internally. Old backup files (v1 flat format) are still supported.
